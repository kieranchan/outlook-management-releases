# Release Naming / 命名规范

## 中文

### Git 标签
- 统一使用语义化版本号：`v主版本.次版本.修订号`
- 示例：`v1.0.0`、`v1.0.1`、`v1.1.0`

### GitHub Release 标题
- 格式：`Outlook Management vX.Y.Z`
- 示例：`Outlook Management v1.0.1`

### 成品压缩包命名
- 新版：`outlook-management-modern-win64-YYYYMMDD.zip`
- 经典版：`outlook-management-classic-win64-YYYYMMDD.zip`

### 命名规则说明
- 全部使用小写英文
- 单词之间使用连字符 `-`
- 平台信息固定为 `win64`
- 日期使用 `YYYYMMDD`
- 不在文件名中混入空格、中文或临时标记

### 推荐示例
- `outlook-management-modern-win64-20260306.zip`
- `outlook-management-classic-win64-20260306.zip`

## English

### Git tag
- Use semantic versioning: `vMAJOR.MINOR.PATCH`
- Examples: `v1.0.0`, `v1.0.1`, `v1.1.0`

### GitHub Release title
- Format: `Outlook Management vX.Y.Z`
- Example: `Outlook Management v1.0.1`

### Packaged zip naming
- Modern edition: `outlook-management-modern-win64-YYYYMMDD.zip`
- Classic edition: `outlook-management-classic-win64-YYYYMMDD.zip`

### Naming rules
- Use lowercase English only
- Separate words with hyphens `-`
- Keep platform segment as `win64`
- Use `YYYYMMDD` for the build date
- Do not include spaces, Chinese characters, or temporary labels in filenames

### Recommended examples
- `outlook-management-modern-win64-20260306.zip`
- `outlook-management-classic-win64-20260306.zip`
